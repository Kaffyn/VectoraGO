PATTERN DETECTION & CODE ANALYSIS PROMPTS
===========================================

Copy these prompts directly into Claude Code.
Replace [PATTERN] and [LANGUAGE] with what you're looking for.

PROMPT 1: ANALYZE FOR PATTERN
===============================

@vectora Analyze this [LANGUAGE] codebase for [PATTERN].

Find:
1. All occurrences of [PATTERN]
2. For each, show:
   - Location (file:line)
   - Code snippet
   - Is it correctly implemented?
   - Risk level (none/low/medium/high)
   - Why it's a risk (if any)

Format as a prioritized list (highest risk first).

LANGUAGE-SPECIFIC PATTERNS
===========================

FOR GO:
--------

PROMPT 1: ANALYZE FOR RACE CONDITIONS
@vectora Analyze this Go codebase for race conditions.

Find:
1. All goroutines (go statements)
2. Shared data accessed by multiple goroutines
3. Synchronization (mutex, channels, atomics)
4. Missing synchronization
5. Risk level for each

PROMPT 1: ANALYZE FOR RESOURCE LEAKS
@vectora Analyze this Go codebase for resource leaks.

Find:
1. All resource opens (file opens, database connections, etc)
2. Are they all closed?
3. What happens if function returns early or panics?
4. Missing defer statements
5. Risk level for each

PROMPT 1: ANALYZE FOR NIL POINTER ERRORS
@vectora Analyze this Go codebase for potential nil pointer errors.

Find:
1. Variables that might be nil
2. Are they checked before use?
3. Return values that might be nil
4. Missing nil checks
5. Risk level for each
